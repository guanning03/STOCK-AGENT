# Trading Agent 汇报叙事与后续实验方案

## 1. 这条叙事是否 make sense

整体上是 make sense 的，而且逻辑是顺的：

`baseline -> 发现 failure modes -> 用更稳定的 workflow 约束去修正`

这条线的优点是：

- 它不是在讲“哪个模型更强”，而是在讲“为什么当前 Trading Agent 不稳定，以及我们如何用轻量级 workflow 改进它”；
- 它把问题拆成了两个很自然的层面：
  - 输出侧问题：动作和仓位控制不稳定；
  - 输入侧问题：多股票、多新闻同时进入时，注意力分配和决策节奏容易失控；
- 两个改进点也正好一一对应：
  - `改进1：动作离散化` 对应输出侧稳定性；
  - `改进2：Top-K` 对应输入侧聚焦与上下文压缩。

不过，为了让这条叙事更严谨，我建议把原来的两句原因表述稍微改一下。

## 2. 建议你这样改写核心问题

### 2.1 原因 1 的更稳写法

你原来写的是：

`LLM 对交易金额控制不敏感，方差过大`

我建议改成：

`连续金额输出自由度过高，LLM 对精确仓位控制不稳定，容易产生幅度过大的微调或不一致调仓。`

这样更好，因为：

- 问题不一定是“模型不敏感”，更可能是“任务形式本身太难”；
- 你真正想表达的是：让模型直接输出 `target_cash_amount` 这种连续数值，控制难度很高；
- 后面的“动作离散化”也就顺理成章成为一个结构性修复，而不是仅仅补丁。

### 2.2 原因 2 的更稳写法

你原来写的是：

`受新闻数据影响，交易次数过多，换手率高`

我建议改成：

`在多股票、新闻密集的日频决策场景下，Agent 容易对短期事件过度响应，导致调仓过频、换手率偏高。`

这样更好，因为：

- 问题不一定是“新闻数据本身有问题”；
- 更准确地说，是“新闻 + 多资产 + 日频决策”这套组合，容易让 Agent 过度反应；
- 这也更自然地引出 `Top-K`：不是否定新闻，而是先减少需要重点处理的股票与事件数量。

如果你后面还想继续扩展，这里甚至可以再分成两个层次：

- `信息过载`：同时处理 20 只股票的价格、持仓和新闻，注意力容易分散；
- `事件过度响应`：面对连续新闻刺激时，模型更容易频繁调仓。

这样会比一句“新闻导致高换手”更有学术味道。

## 3. 建议的汇报主线

### 3.1 Baseline：先说明系统原貌

你可以先把 baseline 讲成一个“能力完整，但约束不足”的系统：

- 在 20 只股票的 universe 上做 batch 决策；
- 输入包括价格序列、新闻、部分基本面和当前持仓状态；
- Agent 需要同时覆盖全部股票；
- 输出是交易方向加连续目标金额；
- 系统按日频进行决策与执行。

这里最重要的一句话可以是：

`baseline 已经具备分析能力，但在输入信息组织和输出动作约束上仍然过于自由，因此策略执行层面不够稳定。`

这句话能自然把后面所有改进串起来。

### 3.2 Baseline 的主要问题：不是“不会分析”，而是“难以稳定执行”

接下来不要急着说 baseline 很差，而是说它暴露出两个 workflow 层面的典型问题：

1. `连续金额输出过于自由`
   - 同样是“看多”，模型可能给出差异很大的目标金额；
   - 很容易出现小幅、频繁、缺乏一致性的调仓；
   - 这会放大执行噪声，也让不同 run 之间方差更大。

2. `多股票新闻上下文过于密集`
   - 每天要同时处理很多股票和多条事件；
   - Agent 容易把注意力花在并不值得立即交易的标的上；
   - 结果是调仓面太宽、交易过频、组合换手过高。

这里建议你把语气写成：

`我们的初步判断是……`

或者：

`从系统机制上看，baseline 很可能存在以下两个 failure modes……`

先不要写成百分之百定论。这样后面补实验会更自然。

### 3.3 改进 1：动作离散化

这一部分你可以定义得非常清楚：

- 不再让模型直接输出连续的 `target_cash_amount`；
- 改为输出少数几个离散 `target_state`；
- 例如：`flat / pilot / core / conviction / hold`；
- 由程序把离散状态自动映射为具体仓位金额。

这一改进的理论作用非常清楚：

- 把高自由度的连续控制问题，改成低自由度的状态选择问题；
- 让模型负责“判断仓位层级”，而不是“精确规划金额”；
- 提高动作一致性、可解释性和可复现性；
- 从机制上抑制无意义的小幅调仓。

一句适合汇报的话是：

`动作离散化的核心不是降低智能，而是把生成式数值控制转化为可执行的状态决策。`

### 3.4 改进 2：Top-K

这一部分建议你把重点放在“注意力预算分配”上，而不是只讲减少 token。

你可以这样讲：

- baseline 中，Agent 每天都要对全部股票进行较重分析；
- 但并不是每只股票每天都值得同等关注；
- 因此在决策前先做一层轻量 shortlist，只让最值得处理的 `Top-K` 股票进入重点决策；
- 未进入 shortlist 的股票默认 `hold` 或保持轻量监控。

这一改进的理论作用是：

- 减少上下文拥挤，提升单只重点股票的分析深度；
- 把有限的推理预算集中到“更可能发生状态变化”的标的；
- 间接减少全市场同时微调的倾向；
- 为后续控制换手率提供结构基础。

一句适合汇报的话是：

`Top-K 的本质不是替模型做决定，而是先决定“哪些股票值得模型认真决策”。`

## 4. 我建议你把这条故事讲成“两个正交改进”

这是我最建议你在汇报时强调的一点：

- `动作离散化` 解决的是输出空间过大、执行不稳定的问题；
- `Top-K` 解决的是输入范围过宽、注意力被稀释的问题。

也就是说，它们不是重复的改进，而是两个正交方向：

- 一个约束输出；
- 一个压缩输入。

这样老师会更容易接受这两个改进并列存在，而且会觉得你的设计是有系统性的。

## 5. 建议的实验方案写法

如果你现在先不放数据，可以直接把后续实验方案写成下面这样。

### 5.1 实验目标

验证轻量级 workflow 优化是否能提升 Trading Agent 的稳定性、可执行性和风险调整后收益。

### 5.2 实验路线

1. `Baseline`
   - 原始 workflow
   - 连续金额输出
   - 全股票日频决策

2. `Baseline + Discrete Actions`
   - 只改动作空间
   - 验证连续金额控制问题是否得到缓解

3. `Baseline + Top-K`
   - 只改输入范围
   - 验证上下文压缩和注意力聚焦是否降低过度调仓

4. `Baseline + Discrete Actions + Top-K`
   - 联合改进
   - 验证输入侧与输出侧约束是否具有互补性

### 5.3 重点看哪些指标

建议先盯这几类指标：

- 收益类：`cum_return`, `cagr`
- 风险类：`max_drawdown`, `sharpe`, `sortino`
- 交易行为类：`trades_count`, `trades_notional`, 平均持仓数, 现金比例
- 稳定性类：不同 run 之间结果波动, 单笔交易金额分布, 小额交易占比
- 效率类：prompt token、耗时、重点分析股票数量

### 5.4 如果老师追问“为什么这两个改进合理”

你可以直接回答：

- `动作离散化` 是为了降低 LLM 在数值控制上的负担，把问题变成更适合语言模型的分类式决策；
- `Top-K` 是为了避免模型在大范围股票池和高密度新闻中分散注意力，把推理资源集中到更重要的资产上；
- 两者分别作用于输出端和输入端，因此具有互补性。

## 6. 我对这条叙事的具体建议

### 6.1 现在就可以讲的版本

你现在完全可以先讲：

`我们不是直接追求更复杂的 Agent，而是先定位 baseline 在 workflow 上的两个主要失稳点，然后分别从动作空间和候选集管理两个方面做轻量改进。`

这个版本很稳。

### 6.2 暂时不要讲得太死的地方

在没有系统数据之前，下面这些说法最好保守一点：

- 不要直接说“新闻导致高换手”，而说“新闻密集场景下更容易过度响应”；
- 不要直接说“离散化一定提升收益”，而说“预计会提升稳定性和执行质量”；
- 不要直接说“Top-K 一定降低换手”，而说“预计会减少无效关注和全市场同步微调”。

### 6.3 如果你想让整条线更完整

后面其实还可以顺势接出第三步：

- `改进3：降低决策频率 / 加 risk gate`

因为如果后续你发现：

- 离散化解决了金额问题；
- Top-K 缩小了关注范围；
- 但交易还是偏频繁；

那么“频率控制”就会成为一个非常自然的下一步，而不是临时补丁。

## 7. 一版可以直接口头汇报的总结

我们当前的 baseline 已经具备了多股票、多信息源的分析能力，但它在 workflow 上仍有两个明显薄弱点。第一，连续金额输出自由度太高，LLM 需要直接控制目标仓位金额，这会导致动作不稳定、微调过多和 run 间方差偏大。第二，在 20 只股票和新闻密集的日频场景下，模型容易对过多标的同时做出响应，造成注意力分散和换手率偏高。基于这两个判断，我们提出两项轻量改进：其一是把连续仓位输出改成离散目标仓位状态，以增强动作的稳定性与可执行性；其二是在正式决策前加入 Top-K shortlist，把有限的推理预算集中到最值得处理的股票上。整体上，这是一条从 baseline failure modes 出发、分别约束输出空间和压缩输入空间的 workflow 优化路径。

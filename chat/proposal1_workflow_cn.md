# 基于轻量级 Workflow 优化的 Trading Agent 研究方案

## 1. 研究主题

本文拟基于现有 `StockBench` 框架，研究一种面向股票交易任务的轻量级 Agent Workflow 优化方法。研究重点不是重新设计一个复杂的多智能体系统，也不是单纯比较不同大模型谁更强，而是关注：

**在尽量不增加系统复杂度和推理成本的前提下，能否通过改进信息组织方式、决策流程和执行约束，提高 Trading Agent 的稳定性、可执行性与风险调整收益。**

---

## 2. 研究背景

当前基于大语言模型的交易系统，通常已经具备一定的分析和推理能力，但在真实交易任务中仍存在一些典型问题：

1. 候选股票较多时，模型注意力容易分散；
2. 原始新闻文本较长，信息密度不均，容易引入噪声；
3. 自由形式的仓位输出不稳定，容易出现算术错误或格式错误；
4. 决策结果缺少统一的硬性风控约束，可能导致极端调仓行为；
5. 即使模型“分析得对”，也不一定能输出“适合执行”的交易动作。

因此，本研究的基本判断是：

**Trading Agent 的问题不完全在于模型能力不足，更关键的问题在于 workflow 设计不够稳定。**

也就是说，若能在输入组织、候选筛选、动作空间和执行前检查等环节做轻量优化，就可能在不显著增加 token 成本的前提下，获得更好的实际交易表现。

---

## 3. 核心思路

本研究保留当前双 Agent 主体结构，只对其 workflow 做轻量改造。整体原则是：

- 不推翻现有回测框架；
- 不引入过重的多层 Agent 协调；
- 尽量复用当前特征、回测和评估模块；
- 重点优化“信息进入模型之前”和“模型输出之后”的关键环节。

本方案的核心逻辑可以概括为一句话：

**不是让 Agent 想得更复杂，而是让 Agent 在更干净的信息流和更稳定的决策约束下工作。**

---

## 4. 改进后的 Workflow

改进后的 Trading Agent Workflow 可分为六个步骤：

1. `Market Overview`：浏览全部候选股票的价格、持仓和新闻概况；
2. `Top-K Shortlisting`：先用低成本规则筛出少数重点股票；
3. `Structured News Analysis`：将原始新闻压缩为结构化事件卡片；
4. `Discrete Decision Generation`：由 Agent 输出离散化交易动作；
5. `Risk Gate`：执行前通过规则进行风控检查；
6. `Execution`：将动作映射为具体仓位变化并送入回测执行。

这一流程的设计目标不是增加更多“智能”，而是逐步压缩错误传播路径。

---

## 5. 四个关键模块的逻辑

### 5.1 Top-K 候选股票粗筛

该模块主要解决“股票太多，模型注意力被稀释”的问题。

在原始流程中，Agent 可能需要同时面对 20 只股票，容易出现以下现象：

- 对真正重要的标的分析不够深入；
- 把上下文浪费在短期内无需调仓的股票上；
- 多资产同时决策时输出变得发散、不稳定。

因此，本研究在深入分析之前，先加入一层廉价筛选，只保留 Top-K（例如 3 或 5 只）股票进入后续深度分析。粗筛信号可以来自：

- 当前是否已有持仓；
- 最近新闻方向是否明显偏多或偏空；
- 价格是否接近短期高点或低点；
- 波动和成交是否出现异常；
- 当前仓位是否已经偏离目标状态。

一个更容易落地的实现方式是：先为每只股票构造一个轻量级 urgency score，再按分数选出 Top-K。该分数不需要额外调用 LLM，而是直接由程序基于以下信号线性加权得到：

- 当前是否已有持仓，以及当前仓位占组合资产的比例；
- 最近 5 条新闻数量，以及其中是否存在明显偏多或偏空关键词；
- 最近 3 至 7 个交易日的价格变动幅度；
- 最近一周收益波动率；
- 当前价格是否贴近 7 日区间高点或低点；
- 最近是否出现过 increase / decrease / close 等主动调仓动作；
- 已持有天数是否偏长，是否需要重新审视。

进入 Top-K 的股票继续走 Fundamental Filter 和 Decision Agent；未进入 Top-K 的股票默认执行程序化 `hold`，只保留最基本的仓位维护。这样做的核心不是“替模型做决定”，而是把有限的上下文预算集中在最可能发生状态变化的标的上。

这一模块的本质不是替代 LLM 决策，而是为 LLM 分配有限的注意力预算。

换句话说：

- 粗筛负责回答“哪些股票值得重点看”；
- Agent 负责回答“这些股票应该如何操作”。

一次快速对照回测（`2025-03-03` 至 `2025-03-31`）也说明这一设计是值得的：相较于不做粗筛，`Top-K=3` 与 `Top-K=5` 都显著降低了 prompt token 和运行时长，其中 `K=5` 的收益与回撤表现最好，更适合作为默认设置；`K=3` 虽然进一步压缩了 token，但组合过于收缩，容易错失横截面机会。

---

### 5.2 新闻压缩为结构化 Event Cards

该模块主要解决“新闻太长、太杂、token 消耗过高”的问题。

原始新闻直接输入模型存在几个不足：

- 多条新闻常常表达重复信息；
- 新闻写法差异大，会干扰模型抽取真正重要的事件；
- 原始文本上下文过长，不利于多股票横向比较；
- token 成本较高。

因此，本研究不直接输入多条原始新闻，而是先将其压缩为结构化的 `event cards`。每条事件卡片可包含以下字段：

- `event_type`：事件类型，如财报、并购、宏观、法律风险、指引调整等；
- `direction`：方向，如 bullish / bearish / neutral；
- `horizon`：影响周期，如 short-term / medium-term；
- `summary`：一句话事件摘要。

这样做的逻辑在于：

**把原始自然语言新闻转化为更适合模型比较和推理的标准化输入。**

这一步既能降低 token 消耗，也能减少文本噪声，使 Agent 更聚焦于“事件本身”，而不是被新闻写作风格影响。

---

### 5.3 自由仓位输出改为离散目标仓位状态

该模块主要解决“模型输出不稳定、执行误差大”的问题。

若让 Agent 直接输出自由形式的目标仓位或金额，容易出现以下情况：

- 目标金额与当前持仓逻辑矛盾；
- 算术错误或格式不规范；
- 相近市场情境下输出波动较大；
- 不同模型之间难以公平比较；
- 模型容易对短期噪声做过度响应，导致高换手和收益被交易噪声侵蚀。

参考当前仓库已有的一年期 `wandb` 回测结果，可以观察到一个较清晰的现象：

- 表现较差的 run 往往交易笔数很高、长期接近满仓持有 20 只股票、组合更分散；
- 表现较好的 run 往往交易次数更少、持仓数量更少、组合更集中，并保留一定现金缓冲；
- 这说明问题不只是“买卖方向对不对”，更在于 Agent 是否会过度频繁地做小幅调仓。

因此，本研究不再采用“模型自由输出金额”的方式，也不采用简单的对称增减档位，而是将动作空间设计为**离散目标仓位状态**。建议定义为：

- `flat`：目标仓位为 `0%`
- `pilot`：目标仓位为 `4%`
- `core`：目标仓位为 `8%`
- `conviction`：目标仓位为 `15%`
- `hold`：保持当前仓位不变

其执行逻辑为：

- Agent 不再直接输出 `target_cash_amount`；
- Agent 只需判断某只股票当前应处于哪一个标准仓位状态；
- 程序再根据当前仓位与目标状态，自动计算应买入或卖出的金额与股数。

例如：

- 当前空仓，输出 `pilot`，则建仓到组合权重约 `4%`；
- 当前仓位约 `4%`，输出 `core`，则继续加仓到约 `8%`；
- 当前仓位约 `15%`，输出 `flat`，则直接清仓；
- 当前仓位已经合适，输出 `hold`，则不做调整。

这种设计相较于“`buy / sell` 对称微调”的优势在于：

- 它保留了原系统“以目标仓位为中心”的核心思想；
- 但把高自由度的连续金额输出，压缩为少数几个可解释、可复现的状态；
- 对高置信机会可以一步进入较高仓位；
- 对明显失效或风险上升的仓位可以一步退出，不需要多日连续减仓；
- 更符合已有 `wandb` 结果所揭示的“低换手 + 适度集中”盈利模式。

这种设计的核心逻辑是：

**把高自由度的生成问题，转化为低自由度的可控决策问题。**

这样做之后，模型不需要直接负责复杂的数值规划，而只需判断当前更接近哪一种标准仓位状态。这样不仅能提高输出一致性，也更便于后续统计和比较。

---

### 5.4 执行前加入 Rule-Based Risk Gate

该模块主要解决“缺少最终风控兜底”的问题。

即使 Agent 分析正确，也可能给出不适合执行的交易建议，例如：

- 单只股票仓位过高；
- 单日总换手率过大；
- 已经浮亏较大却继续加仓；
- 行业或风格暴露过于集中。

因此，本研究在执行前加入一个轻量的规则化风控层。该层不参与分析，只负责检查和约束最终动作，例如：

- 单只股票最大权重不超过 15%；
- 同时持仓股票数量控制在 8 至 10 只；
- 组合保留至少 10% 现金缓冲；
- 单日总换手率不超过 15%；
- 浮亏超过阈值时禁止继续加仓；
- 行业集中度不超过预设上限。

这个模块的作用不是替代 Agent，而是构成一个安全边界。

也就是说：

- Agent 负责提出“想做什么”；
- Risk Gate 负责检查“是否允许这样做”。

这种“模型决策 + 硬规则兜底”的设计非常符合真实交易系统的工程逻辑，也能显著增强论文中的合理性和说服力。

---

## 6. 方案整体逻辑

上述四个模块并不是彼此孤立的，它们构成了一个逐层收缩错误的流程：

1. `Top-K`：减少无关股票，缩小输入范围；
2. `Event Cards`：压缩新闻信息，降低文本噪声；
3. `Discrete Actions`：规范输出形式，降低决策和执行误差；
4. `Risk Gate`：在执行前进行硬约束，降低组合风险。

因此，这个方案不是若干小技巧的简单叠加，而是一套具有明确内部逻辑的 workflow 优化方法。

其总体目标是：

**先减少无效信息，再提升关键信息质量，再规范动作空间，最后控制执行风险。**

## 7. 研究问题

本研究可围绕以下问题展开：

### RQ1

在不显著增加推理成本的前提下，轻量级 workflow 优化是否能够提升 Trading Agent 的收益稳定性？

### RQ2

结构化新闻压缩是否能够在降低 token 消耗的同时保留有效市场信息？

### RQ3

离散动作空间是否能够减少 schema error、算术错误和不稳定输出？

### RQ4

规则化风控层是否能够改善最大回撤与风险调整收益？

---

## 8. 研究假设

为便于实验设计，可以进一步提出如下研究假设：

- `H1`：Top-K 粗筛能降低多资产场景下的决策噪声；
- `H2`：结构化事件卡片能在减少 token 成本的同时保持新闻信号有效性；
- `H3`：离散动作空间能显著提升输出稳定性与执行一致性；
- `H4`：Risk Gate 能有效抑制极端调仓行为，改善最大回撤；
- `H5`：完整的轻量级 workflow 在综合表现上优于原始 baseline。

## 9. 实验设计建议

### 9.1 Baseline

- 原始 StockBench 双 Agent Workflow

### 9.2 对比方案

- Baseline + Top-K Shortlisting
- Baseline + Event Card Compression
- Baseline + Discrete Actions
- Baseline + Risk Gate
- Full Lightweight Workflow

### 9.3 离散动作的具体设定

本研究建议将离散动作具体实现为一组目标仓位状态，而非简单的买卖增量：

- `flat = 0%`
- `pilot = 4%`
- `core = 8%`
- `conviction = 15%`
- `hold = keep current weight`

其中：

- `pilot` 用于低成本试仓或新信号确认初期；
- `core` 用于中等置信度的标准持仓；
- `conviction` 用于高置信机会的集中表达；
- `flat` 用于退出或放弃持仓；
- `hold` 用于降低无效调仓和交易噪声。

执行时由程序将目标状态自动映射为：

- 目标组合权重；
- 目标持仓金额；
- 对应股数与买卖方向。

这样既继承了原系统“输出目标仓位”的结构优点，又显著收缩了模型的自由度。

### 9.4 组合层面的配套设置

为了使离散动作真正服务于一年期收益提升，而不是只改善格式稳定性，本研究建议将其与以下组合层面设定联合使用：

- `Top-K` 粗筛后，只允许少数重点股票进入深度决策；
- 最终同时持仓数控制在 8 至 10 只，而非默认铺满 20 只；
- 保留约 10% 现金缓冲，用于降低被动满仓和噪声交易；
- 将 `hold` 作为默认动作，只有在事件或趋势信号明显变化时才切换状态；
- 用 `flat -> pilot -> core -> conviction` 的状态跃迁表达仓位升级，用 `conviction -> flat` 或 `core -> flat` 表达快速退出。

这一设计更贴近已有回测结果中“少动、集中、让赢家跑”的有效模式。

### 9.5 评估指标

建议从收益、风险、成本和稳定性四个维度评估：

#### 收益与风险

- Cumulative Return
- Maximum Drawdown
- Sortino Ratio
- Sharpe Ratio

#### 稳定性与执行质量

- Schema Error Rate
- Arithmetic Error Rate
- Decision Consistency
- Turnover

#### 成本

- Prompt Tokens
- Completion Tokens
- 总推理成本

## 10. 预期贡献

本研究预期贡献主要包括：

1. 提出一套面向 Trading Agent 的轻量级 Workflow 优化方案；
2. 在现有回测框架中验证不同 workflow 模块的实际效果；
3. 证明 Trading Agent 的改进不一定依赖更大的模型，也可以通过更合理的流程设计实现；
4. 为后续基于 LLM 的交易系统研究提供一个可复现、可消融、可扩展的优化方向。

## 11. 适合作为本科毕设的原因

本方案特别适合作为本科毕业设计，原因主要有三点：

### 11.1 问题边界清楚

研究目标不是泛泛地讨论“LLM 能不能炒股”，而是聚焦于：

**Workflow 优化是否能提升 Agent 的稳定性、可执行性和风险收益表现。**

这一问题规模适中，容易形成完整论文结构。

### 11.2 工程实现可控

该方案建立在现有框架基础上，不需要完全重写系统。研究工作可以集中在：

- prompt 输入组织；
- 特征压缩；
- 动作映射；
- 规则校验；
- 回测评估。

因此整体实现难度适中，适合本科阶段完成。

### 11.3 结果易于分析

即使最终总收益提升有限，也仍然可能在以下维度得到清晰结论：

- token 成本下降；
- 输出更规范；
- 错误率降低；
- 最大回撤改善；
- 风险调整收益提升。

这意味着论文不必把“赚得最多”作为唯一成功标准，更适合学术表达。

---

## 12. 总结

本方案的本质不是让 Agent 变得“更聪明”，而是让 Agent 在一个更合理、更稳定的 workflow 中运行，从而减少错误、提升执行质量并改善风险控制表现。

因此，本研究的核心价值在于：

**通过轻量级流程优化，而非单纯堆叠模型复杂度，提升 Trading Agent 在真实交易环境中的实际可用性。**
